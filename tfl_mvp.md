# THE FINAL LANDING – 1-YEAR MVP

## MVP Promise

A small group of colonists crash on an alien planet.  
The player, an AI, influences where people live, work, and relax.  
From that alone, stories emerge.

If this works, the rest is just expansion.

## CORE PILLAR (ONLY ONE)
🧠 Indirect Control Through Space

Buildings + proximity + time = relationships

That’s it.  
No generations. No children. No diplomacy. No DLC hooks. No epic tech tree.

If this isn’t fun, the game is dead.

## THE LOOP (This Must Be Fun)

1. **Observe 👁️**

   Colonists have:
   - Name
   - Portrait
   - 1 trait (e.g. Loner, Social, Stubborn)
   - 1 preferred role (Explorer, Builder, Cook)

   They move, work, eat, rest.

   You never control them directly.

2. **Influence 🧲**

   You can:
   - Place buildings
   - Assign suggested jobs (not forced)
   - Adjust priorities globally (e.g. "Exploration High")

   That’s all.

   No clicking colonists to command them.

3. **Relationships Emerge ❤️⚡**

   Colonists form opinions based on:
   - Living near someone
   - Working with someone
   - Eating together
   - Stress and fatigue

   **Example:**

   "Mara and Ilya share a workshop. They talk. They clash. They remember."

   Relationships affect:
   - Job success chance
   - Mood
   - Willingness to cooperate

4. **The AI Logs It 📓**

   This is critical.

   The game surfaces meaning through writing, not cutscenes.

   End-of-day AI summary:
   - "Tension increased between Mara and Ilya."
   - "Ilya avoided the Mess Hall tonight."
   - "The colony feels fragmented."

   The player reads the story they accidentally wrote.

## MVP SYSTEMS (STRIPPED TO THE BONE)

1. **Colonists**

   Each colonist has:
   - Name
   - Portrait
   - One personality trait
   - One job preference
   - Relationship values with others (-50 to +50, hidden numbers)

   No stats. No skills. No leveling.

2. **Buildings (Only 5)**

   | Building         | Purpose                  |
   |------------------|--------------------------|
   | Habitat          | Sleeping, recovery       |
   | Mess Hall        | Eating, social           |
   | Workshop         | Building                 |
   | Storage          | Resource buffer          |
   | Exploration Gate | Sends colonists out      |

   That’s it.

   No farms. No hospitals. No defenses.

3. **Resources (Only 2)**

   - Supplies (food + basic materials combined)
   - Salvage (used to build)

   If you need more resource types, the MVP is already failing.

4. **Jobs (Soft Assignments)**

   You can suggest:
   - Explorer
   - Builder
   - Cook
   - Hauler

   Colonists may refuse if:
   - Tired
   - Upset
   - Dislikes coworkers

   Refusal is feedback.

5. **Exploration (Abstracted)**

   - Explorer leaves colony.
   - Timer runs.
   - Returns with:
     - Supplies
     - Salvage
     - A short log entry

   No map. No combat. No player control.

## WHAT MAKES IT FUN (THE REAL TEST)

The MVP succeeds if players:

- Rearrange buildings to "fix" social problems
- Feel clever when relationships improve
- Feel responsible when things fall apart
- Read logs voluntarily

The MVP fails if:

- Players optimize layouts without caring about people
- Logs feel repetitive
- Colonists feel interchangeable

## 12-MONTH MVP TIMELINE (REALISTIC)

### Months 1–2: Skeleton
- Grid map
- Colonists walking
- Place buildings
- Time system

Ugly is fine.

### Months 3–4: Relationships
- Relationship tracking
- Proximity influence
- Mood changes
- Simple refusal logic

This is the core work.

### Months 5–6: Logs & Feedback
- AI daily summaries
- Event-based log entries
- Relationship surfacing in UI

If this isn’t compelling, stop here.

### Months 7–8: Exploration
- Abstract exploration system
- Return events
- Risk vs reward tuning

### Months 9–10: UX & Polish
- Clear UI language
- Tooltips that explain why things happen
- Speed controls
- Pause and inspect

### Months 11–12: Playtest & Kill Features
- External testers
- Cut anything players ignore
- Rewrite logs for tone
